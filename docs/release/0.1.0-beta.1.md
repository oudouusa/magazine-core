# magazine-core 0.1.0-beta.1 release notes

magazine-core is a protocol-first ingestion core for publication metadata: a
Rust host, a language-independent stdio plugin protocol, a canonical SQLite
schema, and a Python plugin SDK. This is the first public beta.

## What you get

- **Rust host + stdio protocol v1** — run plugins as subprocesses that speak a
  framed JSON-RPC protocol over stdin/stdout (`protocol_version = 1`).
- **Canonical SQLite schema + minimal ingestor** — `record_schema_version = 1`.
- **Safe `host_fetch` broker** — generic network safety (http/https only,
  allowed-domains, redirect re-validation, SSRF protection, timeouts, body cap,
  header redaction). Not an evasion layer.
- **DB-backed typed `state_query`** for skip-state and incremental runs.
- **Python SDK** (`magazine_core_plugin_sdk`) with a frozen stable author API.
- **Synthetic examples + conformance fixtures** (Rust/Python byte-parity).

## Plugin trust model (read this)

Plugins are **trusted executable code**. Installing and running a plugin grants
it the permissions of the host process. Subprocess isolation separates lifecycle
and crashes; it is **not** a sandbox. **Do not run untrusted plugins.** See
`SECURITY.md`.

## Install

Build the host from source:

```bash
cargo build --release -p mh-cli --locked
./target/release/mh --help
```

Or use the prebuilt binary for your platform from the GitHub Release
(`magazine-core-mh-linux-x86_64.tar.gz`, `magazine-core-mh-macos-arm64.tar.gz`).

Install the Python SDK from the canonical wheel attached to the Release:

```bash
pip install magazine_core_plugin_sdk-0.1.0-py3-none-any.whl
```

Quickstart:

```bash
mh init-db ./scratch.db
mh inspect ./scratch.db
mh discover ./scratch.db ./plugins.d example
```

## Scope

Included: the generic core only. **Not** included: production site adapters,
anti-bot evasion (proxy/cookie/challenge), credentials, deployment config,
production databases, or downloaded media. To ingest a specific site, write your
own plugin package — the protocol exists for exactly that.

## Verification

Release hardening passes on linux-x86_64 and macos-arm64 (fmt, clippy, `cargo
test`, golden oracle parity, SDK pytest, CLI smoke, binary build, wheel build +
install smoke, CycloneDX SBOM, license inventory, secret scan). The evidence
snapshot is in `docs/release/0.1.0-beta.1-candidate.md`; the run matching this
tag's commit is linked from the GitHub Release, with artifact SHA-256 checksums.

## Known limitations

- The pure-Python wheel is not byte-reproducible across runners, so the Release
  ships a single canonical wheel rather than per-OS wheels.
- Prebuilt binaries cover linux-x86_64 and macos-arm64 only; other platforms
  build from source.
- No 1.0 stability guarantee. Protocol v1 / record_schema v1 / SDK root API are
  frozen for the beta and changes will be evidence-driven.

## Versioning note

Crate and Python package versions stay `0.1.0`; `0.1.0-beta.1` is the release
tag.
